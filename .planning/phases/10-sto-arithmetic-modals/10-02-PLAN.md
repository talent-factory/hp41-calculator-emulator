            // Not an arithmetic op key: fall through to plain STO nn digit accumulator.

The pattern for StoAdd (replicate for StoSub/StoMul/StoDiv with appropriate kind/variant substitution):

```rust
Some(PendingInput::StoAdd(ref acc)) => {
    // Step 3: Y/Z/T/L dispatch to stack registers immediately (no 2-digit accumulation).
    match key.code {
        KeyCode::Char('Y') | KeyCode::Char('y') => {
            self.call_dispatch(Op::StoArithStack {
                kind: StoArithKind::Add,
                stack_reg: StackReg::Y,
            });
            self.pending_input = None;
        }
        KeyCode::Char('Z') | KeyCode::Char('z') => {
            self.call_dispatch(Op::StoArithStack {
                kind: StoArithKind::Add,
                stack_reg: StackReg::Z,
            });
            self.pending_input = None;
        }
        KeyCode::Char('T') | KeyCode::Char('t') => {
            self.call_dispatch(Op::StoArithStack {
                kind: StoArithKind::Add,
                stack_reg: StackReg::T,
            });
            self.pending_input = None;
        }
        KeyCode::Char('L') | KeyCode::Char('l') => {
            self.call_dispatch(Op::StoArithStack {
                kind: StoArithKind::Add,
                stack_reg: StackReg::LastX,
            });
            self.pending_input = None;
        }
        _ => {
            // Not a stack register key: fall through to 2-digit register accumulator.
            self.handle_reg_modal(
                key,
                acc.clone(),
                |reg| Op::StoArith {
                    reg,
                    kind: StoArithKind::Add,
                },
                PendingInput::StoAdd,
            )
        }
    }
}
```

Replicate for StoSub (kind: StoArithKind::Sub, variant: PendingInput::StoSub), StoMul (kind: StoArithKind::Mul, variant: PendingInput::StoMul), StoDiv (kind: StoArithKind::Div, variant: PendingInput::StoDiv).

Note: Accept both upper and lower case letters (Y/y, Z/z, T/t, L/l) to be forgiving of Caps Lock state.

The `_ =>` arm still calls `handle_reg_modal` so that 2-digit numbered registers (00–99) continue to work as before (STOA-01). Esc cancellation is handled inside handle_reg_modal (already implemented per D-08 — no changes needed there).
