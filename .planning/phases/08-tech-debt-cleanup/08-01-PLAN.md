<objective>
Fix the EEX (scientific notation) entry bug in hp41-core. When a user has typed a value like
"1.5e3" into the entry buffer, flush_entry_buf() currently fails with HpError::InvalidOp
because Decimal::from_str() does not parse scientific notation strings. Adding a
.or_else(|_| Decimal::from_scientific(&s)) fallback makes EEX entry functional.

Purpose: Unblock scientific notation entry — the 'e' key appends to entry_buf but the
flush currently rejects the result. This is the root cause of the EEX entry bug.
Output: hp41-core/src/ops/mod.rs with from_scientific fallback + unit test.
</objective>

    fn test_flush_invalid_returns_err() {
        let mut state = make_state_with_entry("notanumber");
