<objective>
Add serde derives to hp41-core's numeric and state types, and migrate `regs` from `[HpNum; 100]` to `Vec<HpNum>`.

Purpose: All downstream persistence work (Plan 03) requires CalcState to implement Serialize + Deserialize. This plan resolves the two primary compilation blockers: (1) serde's built-in fixed-array limit of 32 elements prevents deriving on `[HpNum; 100]`, resolved by switching to `Vec<HpNum>`; (2) HpNum wraps rust_decimal::Decimal which requires the `serde-with-str` feature for lossless string serialization.

Output: Cargo.toml files updated with serde dependencies; num.rs and state.rs compile with full serde derives; existing tests still pass.
</objective>
