The `taiki-e/install-action@v2` step installs `just` — same as in the existing `ci.yml`. The `gui-ci` Justfile recipe (added in Task 1) wraps npm install + tsc + cargo build so no `cargo` is called directly in CI (CLAUDE.md compliance).

```yaml
name: ci-gui

on:
  push:
    branches: [main, develop]
    paths:
      - 'hp41-gui/**'
      - 'hp41-core/**'
  pull_request:
    branches: [main, develop]
    paths:
      - 'hp41-gui/**'
      - 'hp41-core/**'

env:
  CARGO_TERM_COLOR: always
  RUST_BACKTRACE: 1

jobs:
  build:
    name: GUI Build (${{ matrix.os }})
    runs-on: ${{ matrix.os }}
    strategy:
      fail-fast: false
      matrix:
        os: [ubuntu-latest, macos-latest, windows-latest]

    steps:
      - uses: actions/checkout@v4

      - uses: dtolnay/rust-toolchain@stable

      - uses: Swatinem/rust-cache@v2
        with:
          workspaces: hp41-gui/src-tauri -> hp41-gui/src-tauri/target

      - uses: actions/setup-node@v4
        with:
          node-version: 'lts/*'

      - uses: taiki-e/install-action@v2
        with:
          tool: just

      - name: Install Linux system deps
        if: matrix.os == 'ubuntu-latest'
        run: sudo apt-get update && sudo apt-get install -y libwebkit2gtk-4.1-dev libappindicator3-dev librsvg2-dev patchelf

      - name: GUI CI
        run: just gui-ci
```

Notes:
- `Swatinem/rust-cache@v2` `workspaces` key: use `hp41-gui/src-tauri -> hp41-gui/src-tauri/target` to cache the GUI crate's target directory separately from any root workspace cache.
- The Linux WebKit apt-get step stays in YAML (not in `just gui-ci`) because it requires the `if: matrix.os == 'ubuntu-latest'` conditional guard.
- `just gui-ci` handles: `npm install` (in hp41-gui), `npx tsc --noEmit` (in hp41-gui), `cargo build --release` (via `--manifest-path hp41-gui/src-tauri/Cargo.toml`).
