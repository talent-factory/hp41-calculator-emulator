## Spacing Scale

The project uses an 8-point scale. Values in use across the existing
.calculator, .print-panel, and .stack-panel components:

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Panel header vertical padding (4px top/bottom) |
| sm | 8px | Panel header horizontal padding; step row horizontal padding |
| sm+ | 6px | Step list container vertical padding (matches print-panel-content) |
| md | 16px | .calculator top/bottom margin (inherited, no change) |
| lg | 24px | Not used in this panel |
| xl | 32px | Not used in this panel |
| 2xl | 48px | Not used in this panel |
| 3xl | 64px | Not used in this panel |

Exceptions: 6px vertical padding on .prgm-panel-content matches the
existing .print-panel-content (6px 8px) — a deliberate carry-over for
visual consistency, not a deviation.

Step row height: 1px top + 1px bottom padding per row, line-height 1.4.
At 12px font size this yields ~18.8px per row — approximately 8 rows
visible in the 160px max-height panel.

## Typography

| Role | Font family | Size | Weight | Line Height | Usage |
|------|-------------|------|--------|-------------|-------|
| Step label | 'Courier New', Courier, monospace | 12px | 400 (regular) | 1.4 | Each program step row (e.g. "001 STO 00") |
| Panel header | 'Courier New', Courier, monospace | 10px | 400 (regular) | 1.0 | "PRGM — N steps" header |

Notes:
- 12px step labels match the 11px/13px range of the existing print
  panel (11px) and stack panel (13px); 12px splits the difference and
  improves readability at the wider character count of step strings.
- 10px header matches .print-panel-header exactly (App.css line 110).
- letter-spacing: 0.1em on header (matches .print-panel-header).
- No heading or display roles — this panel is data-only.
- Two weights only: 400 regular for all text; the active-step
  highlight uses background color, not bold weight, to avoid layout
  reflow when the highlight moves.

## Color

All values derived from existing App.css to maintain the dark
calculator aesthetic (CONTEXT.md "Claude's Discretion").

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #0d0d0d | .calculator background (inherited) |
| Secondary (30%) | #1a1a1a | .prgm-panel background; step list background |
| Panel chrome | #252525 | .prgm-panel-header background (matches .print-panel-header) |
| Border / separator | #3a3a3a | border-top separating panel from keyboard; header border-bottom |
| Step text | #c8c8c8 | Default step row text (matches .print-line color) |
| Header text | #888 | "PRGM — N steps" label (matches .print-panel-header color) |
| Active step bg | #1e3a1e | Highlight background for the current pc step row |
| Active step text | #c8e6c9 | Text color on highlighted step (matches .display color — green LCD tone) |
| Inactive step alt | none | No alternating row shading — keeps visual weight low; highlight contrast alone is sufficient |

Accent reserved for: active step highlight row only (.step-active
class). No other element in this panel uses the accent green.

Destructive: not applicable — this panel has no destructive actions.

Color rationale for active step (#1e3a1e / #c8e6c9):
- The dark green background (#1e3a1e) is low-contrast against
  the panel background (#1a1a1a) for non-highlighted rows, high-
  contrast for the highlighted row — the HP-41 green LCD aesthetic.
- #c8e6c9 is already used in .display; reusing it here ties the
  active step visually to the calculator display.

## Interaction Contract

### Auto-show / hide

- Panel renders when `calcState.annunciators.prgm === true`.
- Panel is absent from the DOM when `prgm` is false (not hidden with
  display:none — conditional JSX).
- Source: D-08 (locked).

### Auto-scroll to active step

- `useRef<HTMLDivElement>(null)` named `activeStepRef`.
- The step div at index `calcState.pc` receives `ref={activeStepRef}`.
- `useEffect` watches `calcState?.pc`; fires
  `activeStepRef.current?.scrollIntoView({ behavior: 'smooth', block: 'nearest' })`.
- Smooth scroll behavior prevents jarring jumps on rapid SST/BST.
- Source: D-09 (locked).

### SST / BST keyboard and click routing

| Trigger | Key ID | Tauri command | Guard |
|---------|--------|---------------|-------|
| F7 key | 'sst' | sst_step | busyRef.current check |
| F8 key | 'bst' | bst_step | busyRef.current check |
| SVG SST click | 'sst' | sst_step | busyRef.current check |
| SVG BST click | 'bst' | bst_step | busyRef.current check |

- `handleClick` branches on keyId before the default `dispatch_op` call.
- `resolveKeyId()` maps 'F7' → 'sst' and 'F8' → 'bst'.
- Source: D-06, D-07 (locked).

### Window height

- `tauri.conf.json` `windows[0].height`: 700 → 900.
- No UI interaction required; the taller window accommodates the panel.
- Source: D-10 (locked).

### Empty program state

- When program is empty, `program_steps` always contains `["000 END"]`
  (from `format_all_steps()` in prgm_display.rs).
- Panel still renders with one row: "000 END".
- Header reads "PRGM — 1 step" (singular).
- `pc` is 0, so "000 END" row is highlighted.

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Panel header (empty program) | PRGM — 1 STEP | Uppercase from text-transform; singular |
| Panel header (N steps) | PRGM — N STEPS | N = program_steps.length |
| Panel header (plural check) | PRGM — N STEPS | Always plural for N >= 2 |
| Empty state | "000 END" | The only row; no separate empty state message needed |
| Error state | n/a | Panel has no error states; SST/BST are safe at boundaries |
| Destructive confirmation | n/a | No destructive actions in this panel |
| SST key label | SST | Existing Keyboard.tsx label, unchanged |
| BST key label | BST | Existing Keyboard.tsx label, unchanged |

Header copy implementation note: the JSX expression is
`PRGM — {count} {count === 1 ? 'STEP' : 'STEPS'}`. The em dash (—)
is a Unicode literal, not a double hyphen. Uppercase is applied by
CSS `text-transform: uppercase`, so the JSX string can use lowercase
if preferred; the rendered output is always uppercase.

## Responsive / Constraint Notes

- `.calculator` is fixed at 392px width — no responsive breakpoints.
- Panel inherits this width via `width: 100%`.
- max-height: 160px chosen in D-09 — approximately 8 step rows at
  12px/1.4 line-height. Overflow scrolls vertically.
- Window height 900px provides ~200px of space below the keyboard for
  the panel plus the print panel (if both visible simultaneously).
  At 160px max-height + 6px padding top + bottom + 1px borders, the
  prgm-panel occupies at most ~175px.
