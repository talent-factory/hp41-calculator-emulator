<objective>
Wire the GUI-side cancellation channel for long-running Math Pac I ops (INTG / SOLVE / DIFEQ). Phase 28 already shipped the `cancel_requested: Arc<AtomicBool>` field on `CalcState` AND the per-64-samples polling loop inside `op_integ`/`op_solve`/`op_difeq`. Phase 31-02 closes the loop by (a) adding a `request_cancel` Tauri command that flips the AtomicBool WITHOUT acquiring the AppState Mutex (Pitfall 1 — Mutex is held by the active dispatch_op for the entire INTG run), (b) adding the matching permission TOML + capabilities entry, (c) authoring `scripts/check-tauri-permissions.sh` (a CI gate that doesn't currently exist per RESEARCH Open Q1), (d) adding the workflow-opener reset of `cancel_requested` so cancellation is idempotent across runs (per Open Q6 audit verified by `grep -rn "cancel_requested.store(false" hp41-core/src/ops/math1/` returning ZERO matches against the current tree — all three programs lack the reset and Plan 31-02 adds it).

Purpose: Make long-running Math Pac I computations cancellable from the GUI in <100ms, deadlock-free under the AppState-Mutex-held-by-dispatch model. GUI-05 acceptance.

Output: One new Tauri command (`request_cancel`), one new permission TOML, one new CI gate script, capabilities/default.json updated, two integration tests, one hp41-core test, and (per Open Q6 audit confirmed) three one-line `cancel_requested.store(false, Relaxed)` insertions in `hp41-core/src/ops/math1/{integ,solve,difeq}.rs` workflow openers.
