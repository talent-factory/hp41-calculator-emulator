<objective>
Wire the shared `submit_modal`, `cancel_modal`, and `submit_modal_with_label` core functions into Tauri-command thunks so the GUI can drive Math Pac I modal workflows. The XEQ-by-name modal in `App.tsx` already exists from v2.1 and routes `xeq_SINH`/`xeq_MATRIX`/etc. through `dispatch_op` → `key_map::resolve` → `Op::Xeq(name)` → core's resolver chain → `xrom_resolve` (Phase 29 already wired). Plan 31-03's job is therefore (a) adding the 3 new Tauri thunks for the modal-workflow actions that 31-05's frontend state machine will dispatch, (b) projecting 4 missing fields onto CalcStateView so the frontend can make state-aware routing decisions (Pitfall 10), (c) overriding `HpError::Canceled` → `"CANCELED"` uppercase per UI-SPEC (Pitfall 4), (d) authoring a D-25.6 parity regression test that asserts the CLI-resolved and GUI-equivalent paths produce identical X-register output for at least 3 Math Pac I functions (e.g., SINH, ASINH, TANH).

Purpose: GUI-02 acceptance. Establish the IPC surface 31-05 builds the R/S 3-way and Esc-cascade routing on top of. Lock D-25.6 parity behind a regression test.

Output: 3 new Tauri command thunks (~15 lines total of Rust glue), 3 new permission TOMLs, capabilities/default.json updated, types.rs extended with 4 projected fields + 1 From-impl override, App.tsx TypeScript CalcStateView interface mirror extended, 1 new integration-test file.
</objective>
