## Tech Debt: 8 items across 3 areas

### Area 1: EEX / entry_buf (Phase 04 carryover)
- EEX key appends 'e' to entry_buf but `Decimal::from_str` does not parse scientific notation — EEX entry fails silently
- entry_buf accepts duplicate '.' and 'e' characters with no guard

### Area 2: Keyboard coverage gaps (Phase 04/05 carryover)
- SIN (Op::Sin) has no interactive key — 'S' was reassigned to STO modal in Phase 5
- STO+/−/×/÷ keyboard modals defined but never triggered — `PendingInput::StoAdd/Sub/Mul/Div` are dead code
- CLREG (Op::Clreg) has no keyboard binding
- AlphaClear (Op::AlphaClear) has no keyboard binding

### Area 3: Documentation staleness
- `help_data.rs:32` incorrectly shows `("S", "SIN", ...)` — 'S' actually opens STO modal
- Phase 02 VERIFICATION.md shows `gaps_found` but bugs are fixed; VERIFICATION.md is stale
