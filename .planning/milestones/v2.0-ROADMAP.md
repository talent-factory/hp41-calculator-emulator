# Milestone v2.0: HP-41 Calculator Emulator Tauri GUI

**Status:** ✅ SHIPPED 2026-05-10
**Phases:** 13–18
**Total Plans:** 19

## Overview

v2.0 added a pixel-perfect Tauri v2 + React + TypeScript desktop GUI (`hp41-gui`) reusing `hp41-core` unchanged alongside the existing `hp41-cli`. The milestone shipped in 2 days (2026-05-09 → 2026-05-10) with 183 files changed, 30,358 insertions, delivering a fully functional HP-41C calculator desktop application with SVG skin, program listing panel, and cross-platform CI.

## Milestone Summary

**Key Decisions:**
- `hp41-gui` is a nested standalone workspace (excluded from root `Cargo.toml` members); Tauri deps never enter root resolver (Phase 13)
- Bundle identifier `ch.talent-factory.hp41` — overrides scaffold default; avoids macOS sandbox/keychain issues (Phase 13)
- Tauri v2.11 app-command permissions: TOML files in `src-tauri/permissions/` required (no auto-generation for inline commands) (Phase 14)
- `CalcStateView` display_str priority: entry_buf → format_alpha(alpha_mode) → format_hpnum(stack.x) — matches hp41-cli logic (Phase 14)
- KEY_DEFS has 44 entries (not 40): HP-41C has 44 key positions (9+8+9+9+9 across 5 rows); ENTER is one entry with colSpan:2 (Phase 16)
- SVG shadow: manual rect over filter — simpler, no GPU compositing overhead (Phase 16)
- `transform-box: fill-box` required for SVG animation — without it, `scale()` transforms from canvas origin rather than key center (Phase 16)
- `handleClick` moved before `handleKey` in App.tsx — React dependency ordering (handleKey depends on handleClick) (Phase 18)
- `format_all_steps` always appends END so `pc==program.len()` highlights correctly (Phase 18)

**Issues Resolved:**
- AppState Mutex not released before disk I/O in auto-save thread (fixed in Phase 17, commit ff39017)
- Tailwind interference with vanilla CSS (removed in Phase 15)
- `e.repeat` duplicate IPC calls in keyboard listener (fixed with `busyRef` debounce in Phase 15)

**Issues Deferred to v2.1:**
- SKIN-04: 14-segment SVG font for authentic LCD display rendering
- SKIN-05: Keyboard shortcut overlay (port `?` help panel from CLI)
- PROG-02: Full keyboard assignment display in USER mode
- `prgm_mode` keyboard binding for 'p' key (currently mapped to `prx`)
