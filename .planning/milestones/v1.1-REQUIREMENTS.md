## v1.1 Requirements (All Shipped)

### INFRA — Infrastructure

- [x] **INFRA-01**: MSRV formally declared at 1.85 with workspace inheritance; `rust_decimal` pinned to 1.42; CI MSRV job added
  - Phase 9, Plan 09-01
  - Outcome: `rust-version = "1.85"` in `Cargo.toml [workspace.package]`; `rust_decimal` 1.41 → 1.42; GitHub Actions MSRV job

### INPUT — EEX Entry Hardware Fidelity

- [x] **INPUT-01**: EEX trailing-e-without-exponent commits as exponent 00 (not silent discard); `flush_entry_buf` normalizes before parse chain
  - Phase 9, Plans 09-02 + 09-03
  - Outcome: `flush_entry_buf` appends "00" before parse; TUI shows `1.5E_ _` exponent placeholder cursor

- [x] **INPUT-02**: Empty-buffer EEX inserts implicit mantissa 1 (HP-41 hardware behavior)
  - Phase 9, Plan 09-03
  - Outcome: EEX on empty buffer shows `1   _` in exponent entry mode

- [x] **INPUT-03**: While in partial-exponent state, TUI shows exponent placeholder cursor
  - Phase 9, Plan 09-03
  - Outcome: `format_entry_buf_display` helper renders 0/1/2 slots with underscore placeholders

### STOA — STO Arithmetic Modals

- [x] **STOA-01**: S → op-key → 2-digit-register: STO+/−/×/÷ for R00–R99
  - Phase 10, Plans 10-01 + 10-02
  - Outcome: `StackReg` enum + `Op::StoArithStack` + `op_sto_arith_stack()` in core; TUI routing in app.rs

- [x] **STOA-02**: S → op-key → Y/Z/T/L: STO+/−/×/÷ for stack registers
  - Phase 10, Plan 10-02
  - Outcome: Y/Z/T/L dispatch in StoAdd/Sub/Mul/Div arms

- [x] **STOA-03**: Esc cancellation at any modal step (S, op key, first digit) leaves no side effects
  - Phase 10, Plans 10-01 + 10-02
  - Outcome: `PendingInput::None` on Escape; no state change

### PRNT — Print Emulation

- [x] **PRNT-01**: PRX writes X in current display format, right-aligned to 24 characters, to console
  - Phase 11, Plans 11-01 + 11-02
  - Outcome: `Op::PRX` in `ops/print.rs`; formatted via `format_hpnum`; right-aligned to 24 chars

- [x] **PRNT-02**: PRA writes ALPHA register contents, left-aligned to 24 characters, to console
  - Phase 11, Plans 11-01 + 11-02
  - Outcome: `Op::PRA` in `ops/print.rs`; left-aligned to 24 chars

- [x] **PRNT-03**: PRSTK writes full stack in hardware order (T, Z, Y, X, LASTX, ALPHA), one line per register
  - Phase 11, Plans 11-01 + 11-02
  - Outcome: `Op::PRSTK` in `ops/print.rs`

- [x] **PRNT-04**: `--print-log <path>` appends all PRX/PRA/PRSTK output to a file
  - Phase 11, Plan 11-02
  - Outcome: `print_log_writer` in hp41-cli; `--print-log` arg via clap

### SYNT — Synthetic Programming

- [x] **SYNT-01**: GETKEY pushes HP-41 row-column key code of last pressed key to X when executed
  - Phase 12, Plans 12-01 + 12-02
  - Outcome: `CalcState.last_key_code`; `keycode_to_hp41_code()` in keys.rs; `Op::GetKey` pushes code to X

- [x] **SYNT-02**: NULL executes without changing any stack register or lift flag (Neutral stack-lift)
  - Phase 12, Plan 12-01
  - Outcome: `Op::Null` with `LiftEffect::Neutral` in dispatch()

- [x] **SYNT-03**: STO M/N/O and RCL M/N/O work in programs and survive JSON save/load round-trip
  - Phase 12, Plans 12-01 + 12-02
  - Outcome: `reg_m/n/o` fields on `CalcState` with `#[serde(default)]`; M/N/O branches in StoRegister/RclRegister

- [x] **SYNT-04**: Hex-byte insertion modal accepts 2-digit hex code from curated safe subset (23 entries); codes outside subset rejected
  - Phase 12, Plans 12-01 + 12-02
  - Outcome: `HexModal(String)` `PendingInput` variant; `synthetic_byte_to_op()` validates against safe subset; 'X' interceptor (PRGM-mode gated)
