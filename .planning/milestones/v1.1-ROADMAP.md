**Status:** ✅ SHIPPED 2026-05-09

## Overview

v1.1 closed all known gaps in the CLI feature set — hardware-faithful EEX entry, STO arithmetic keyboard modals, PRX/PRA/PRSTK print emulation, and synthetic programming (GETKEY, NULL, hidden registers M/N/O, HexModal). The milestone shipped in 2 days (2026-05-08 → 2026-05-09) with 221 files changed, 17,718 insertions.

## Milestone Summary

**Key Decisions:**
- EEX trailing-e → append "00": hardware fidelity; `flush_entry_buf` normalizes before parse chain (Phase 9)
- Empty-buffer EEX inserts "1e": HP-41 hardware behavior; implicit mantissa (Phase 9)
- `print_buffer.push()` is hp41-core-only; no `println!`/`eprintln!` allowed in `ops/print.rs` — core I/O-free invariant (Phase 11)
- `#[serde(skip)]` on `print_buffer`: transient runtime field; never in JSON save files (Phase 11)
- HexModal byte validation MUST happen before `state.program.insert()` — security invariant T-12-W2-02 (Phase 12)

**Bugfixes Found in Review:**
- `Vec::insert` panic after ISG/DSE skip-at-end (CR-01, fixed in Phase 11 gap closure)
- F5 overwriting `last_key_code` before GETKEY (F5 → 0 in `keycode_to_hp41_code`)

**Issues Deferred:**
- SYNT-05: Full FOCAL byte-code table (~200 codes)
- SYNT-06: GETKEY interrupt-style capture (requires event loop redesign)
- PRNT-05/06: Scrollable print history, ADV/PRREG/TRACE
- STOA-04: STO arithmetic via indirect addressing
- Tauri v2 GUI — deferred to v2.0
