## What Was Built

### Part B: flush_entry_buf normalization (hp41-core/src/ops/mod.rs)

Extended `flush_entry_buf` to normalize two new trailing exponent suffixes before parsing:

- `"1e-"` → `"1e-00"` → parses as 1.0 (exponent −00 = 0)
- `"1e"` → `"1e00"` → parses as 1.0 (already normalized in the combined condition)

Implementation uses a single combined condition:
```rust
if s.ends_with("e-") || s.ends_with("E-") || s.ends_with('e') || s.ends_with('E') {
    s.push_str("00");
}
```

Also added the WR-02 fix: `entry_buf` is now cleared only after successful parse, not before. This preserves user input on parse error.

### Part A: CHS intercept (hp41-cli/src/app.rs)

Added a new branch in `handle_key()` inside the `if let KeyCode::Char(c)` block, after the `'e'` EEX handler:

```rust
if c == 'n' && self.state.entry_buf.contains('e') {
    // Toggle exponent sign in-place — no flush, no dispatch.
    if let Some(e_pos) = self.state.entry_buf.find('e') {
        let after_e = &self.state.entry_buf[e_pos + 1..];
        if after_e.starts_with('-') {
            self.state.entry_buf.remove(e_pos + 1);  // "1e-2" → "1e2"
        } else {
            self.state.entry_buf.insert(e_pos + 1, '-');  // "1e2" → "1e-2"
        }
    }
    self.message = None;
    return;
}
```
